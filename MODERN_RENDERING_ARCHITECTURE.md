## 1. Rendering Pipeline: Forward+

### Why Forward+?
- **Transparency Friendly**: Quake has heavy water, sprites, additive effects
- **Clustered Lighting**: 1024+ dynamic lights per frame
- **Memory Efficient**: No G-buffer (saves 4-8 bytes/pixel vs Deferred)
- **M3 Optimized**: Tile-based deferred GPU loves Forward+

### Pipeline Stages:
```
1. Depth Pre-Pass (Opaque only)
2. Light Culling (Compute Shader - 3D Grid Clustering)
3. Forward Shading (Per-object with clustered lights)
4. Transparency Pass (Depth-sorted, additive/blend)
5. Post-Processing (SSR, Bloom, Tone Mapping, FSR)
```

### Light Clustering Specifications:
- **Grid Size**: 16x8x8 tiles (screen-space XY, view-space Z)
- **Max Lights per Cluster**: 64 (typical: 8-16)
- **Light Types**: Point, Spot, Directional
- **Shadow Maps**: Cascaded for directional, omnidirectional for point

## 3. FSR 2.2 Implementation

### Integration Points:
```c
typedef struct {
    qboolean enabled;
    float qualityMode;        // 0.5=Performance, 0.67=Balanced, 0.77=Quality
    int renderWidth;          // Dynamic resolution
    int renderHeight;
    int displayWidth;         // Native 4K
    int displayHeight;
    
    // Temporal data
    texture_t *motionVectors;
    texture_t *previousFrame;
    texture_t *reactiveMask;
} fsr_state_t;
```

### FSR Pipeline:
1. **Motion Vector Generation** (during forward pass)
2. **Autogenerated Reactive Mask** (transparency detection)
3. **FSR Upscale + Sharpening** (compute shader)
4. **Present to Swapchain**

## 4. Screen Space Reflections (SSR)

### Full-Screen SSR Specification:
- **Resolution**: Dynamic (50%-100% based on performance)
- **Ray Steps**: 64 steps (binary search refinement)
- **Thickness**: 2.0 pixels (geometric tolerance)
- **Fresnel**: Schlick approximation
- **Fallback**: Blur + fade to reflection probe

### SSR Integration:
```glsl
// In Forward+ fragment shader
vec3 ssrReflection = ScreenSpaceReflection(worldPos, normal, viewDir, roughness);
finalColor = mix(ssrReflection, skyboxReflection, fresnel);
```

### Water-Specific Enhancements:
- **Normal Map Distortion**: Animated tangent-space normals
- **Refraction**: Screen-space refraction offset
- **Absorption**: Beer-Lambert depth-based color
- **Caustics**: Projected animated texture (optional)

## 6. Intelligent Optimizations

### M3-Specific Optimizations:
1. **Tile-Based Rendering**: Minimize tile loads/stores
2. **Unified Memory**: Zero-copy texture streaming
3. **Neural Engine**: Optional AI upscaling (future)
4. **ProMotion**: Variable refresh rate (120Hz)

### General Optimizations:
1. **Frustum Culling**: CPU + GPU hardware culling
2. **Occlusion Culling**: Hardware occlusion queries
3. **LOD System**: Distance-based model detail
4. **Texture Streaming**: Virtual texturing (megatextures)
5. **Batch Rendering**: Instanced draw calls
6. **Async Compute**: Light culling parallel to shading
7. **Dynamic Resolution**: Frame-time targeting
8. **Predictive Loading**: Prefetch next map area

### Memory Budget (2GB VRAM):
| Resource | Allocation | Notes |
|----------|------------|-------|
| Textures | 1024 MB    | Compressed BC/DXT/ASTC |
| Geometry | 256 MB     | Vertex/Index buffers |
| Shadows  | 128 MB     | Shadow atlas |
| G-Buffer | 0 MB       | Forward+ doesn't use! |
| Post-Proc| 128 MB     | MRT + temporals |
| FSR      | 64 MB      | Motion vectors, history |
| Reserved | 400 MB     | OS + overhead |

## 8. Build Configuration

### New Makefile Targets:
